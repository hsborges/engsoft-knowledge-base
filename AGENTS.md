# AGENTS.md

## Repository shape (do not guess)

- This is a docs-only Docusaurus repo: source content lives in `docs/`; site wiring is in `docusaurus.config.js` and `sidebars.js`.
- `sidebars.js` pins Home to doc id `README`, so `docs/README.md` must exist with that name.
- Top-level sidebar sections are only `curso`, `legislacao`, and `faq` (all autogenerated from those directories).
- There is no app code beyond theming in `src/css/custom.css`.

## Commands that matter

- Use Node 20+ (`package.json` engines).
- Install dependencies with `npm ci` (CI uses this; lockfile is `package-lock.json`).
- Local docs server: `npm start` (binds `0.0.0.0:3000`).
- Required check before commit: `npm run lint:md` (also enforced by `.husky/pre-commit`).
- Autofix Markdown issues: `npm run lint:md:fix`.
- CI-parity verification: `npm run build`.

## Markdown/lint quirks

- Markdown lint runs on `**/*.md` via `.markdownlint-cli2.jsonc`.
- `MD013` (line length) and `MD041` (first-line heading) are disabled.
- `.opencode/**` is excluded from markdownlint.

## Content conventions that are easy to miss

- FAQ files stay flat in `docs/faq/` (no thematic subfolders), named `<TEMA>.md`, and should keep frontmatter keys `tipo_documento`, `tema`, `base_normativa_principal`, and `ultima_revisao`.
- When adding/renaming FAQ files, update `docs/faq/README.md` index table.
- Most section READMEs maintain manual inventory tables (including `Total de arquivos`); update those counts/tables when catalog content changes.
- Repository content is written in Portuguese (`pt-BR`); keep terminology and frontmatter style consistent.

## Normative-query guardrail (from repo instructions)

- For normative answers, do not treat `docs/faq/` as primary legal source; use original documents under `docs/curso/` and `docs/legislacao/`, citing exact files.

## OpenCode repo config

- `.opencode/opencode.json` sets default agent to `coord`; behavior guidance is mirrored in `.agents/agents/coord.md` and `.opencode/agents/coord.md`.

## Deployment fact

- GitHub Actions deploys on push to `master` (`.github/workflows/deploy-docs.yml`): `npm ci` -> `npm run build` -> publish `build/` to `gh-pages`.
